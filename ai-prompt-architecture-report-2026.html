🤖 主流模型 System Prompt

GPT-4o / GPT-4.1

System Message JSON Mode Structured Output

XML Tags Extended Thinking CLAUDE.md
特点：推崇 XML 标签结构组织 System Prompt，如 <role>、<context>、<instructions>。Claude Code 自动读取项目 CLAUDE.md 注入上下文。支持 Extended Thinking（长思考链），在 prompt 中通过 thinking budget 控制。

System Instruction Safety Config Multimodal

🔧 Agent 框架 Prompt 设计模式
OpenAI Agents SDK OpenAI
模式：Instructions 字符串/函数 + Handoffs + Guardrails
核心设计：每个 Agent 配置 instructions（支持静态字符串或动态函数），通过 handoffs 实现 Agent 间任务移交。Guardrails 分为输入/输出检查，在执行前后拦截。Tracing 内建全链路追踪。

Prompt 注入形式：instructions → 拼入 system prompt → 附带可用 handoff agent 描述 → 附带 guardrails 规则
CrewAI CrewAI Inc.
模式：Role + Goal + Backstory 三段式角色定义
核心设计：每个 Agent 通过 role（角色名）、goal（目标）、backstory（背景故事）三段定义。Task 分配时附 expected_output 格式约束。支持 Sequential（顺序）和 Hierarchical（层级）两种流程。

Prompt 结构："You are {role}. {backstory}. Your goal is {goal}. Current task: {description}. Expected output format: {expected_output}."
LangGraph LangChain
模式：State Graph + Node-level Prompt + 条件路由
核心设计：以有向图定义 Agent 工作流，每个 Node 独立注入 System Prompt。State 对象在节点间传递上下文。通过 Conditional Edges 实现动态路由。Checkpointing 支持状态持久化和人机交互。

Prompt 注入形式：开发者在每个 Node 函数中自行构造 messages（system + user），灵活性最高但需要手动管理 prompt 上下文。
AutoGen Microsoft
模式：ConversableAgent + system_message + 群聊管理
核心设计：每个 ConversableAgent 携带 system_message（角色定义+行为约束），通过 GroupChat + GroupChatManager 管理多 Agent 对话。支持 Code Executor Agent 隔离执行代码。

Prompt 结构："{system_message}\n\nContext: {chat_history}\n\nReply to: {speaker}: {message}" — 对话历史自动注入
Claude Code Anthropic
模式：内建 System Prompt + CLAUDE.md 项目上下文注入
核心设计：Claude Code 自带复杂的 System Prompt（包含工具使用说明、规则、行为约束）。自动读取项目根目录的 CLAUDE.md（或 AGENTS.md）作为项目级 context 注入。支持 Memory 机制跨会话持久化。

Prompt 层级：内建 System > CLAUDE.md > Memory > 用户输入 — 层层叠加
MetaGPT DeepWisdom
模式：SOP-based 角色模板、结构化输出（文档/图表/代码）
核心设计：模拟软件公司 SOP——PM、Architect、Engineer 等角色各有固定的 Prompt 模板。输出为结构化文档（PRD、设计文档、代码）。角色间通过消息总线传递结构化数据。

Prompt 模板：每个角色有预定义的 System Prompt 模板 + 待填充的 context 变量（项目名、需求、上一步输出等）
Qwen-Agent Alibaba
模式：配置化 System Prompt + Function List + Agent Group
核心设计：Agent 配置中包含 system_message + function_list（工具定义）。支持 Agent Group 多 Agent 协作。内建 RAG 检索增强和 Code Interpreter。

Prompt 结构：兼容 ChatML 格式，system message + 可用工具描述 + 用户消息 — 工具调用返回后自动拼接进对话历史

🤝 多 Agent 协作模式

🎯 编排式 (Orchestration)

一个主控 Agent 负责任务分解、分配、结果汇总。代表：OpenAI Agents SDK (Handoffs)、CrewAI (Hierarchical Process)

Prompt 特点：主控 Agent 的 system prompt 包含所有子 Agent 的能力描述和分配规则，类似项目经理的职责说明

🔄 对等式 (Peer-to-Peer)

各 Agent 地位平等，通过消息传递自主协商。代表：AutoGen (GroupChat)、LangGraph (State Graph)

Prompt 特点：每个 Agent 携带自身角色+对话协议规则，无中央控制器，prompt 中包含 "何时发言/何时沉默" 的规则

🏭 流水线式 (Pipeline)

固定流程的链式传递，输出→输入。代表：CrewAI (Sequential)、MetaGPT (SOP)

Prompt 特点：每个阶段 Agent 的 prompt 包含 "你的输入来自上一阶段，你的输出格式要求为..."，形成标准化交接规范
🚀 2025-2026 趋势洞察
1

System Prompt 工程化

从手写 prompt 到结构化配置，Claude Code 的 CLAUDE.md、OpenAI 的 Instruction Hierarchy 代表了 System Prompt 从「艺术」到「工程」的转变

2

Context Window → Context Engineering

100K+ token 上下文窗口普及后，焦点从 "prompt 怎么写" 转向 "哪些上下文该放入 prompt"——精选 > 堆量

3

动态 Prompt 生成

不再写死 prompt，而是根据任务动态组装——OpenAI Agents SDK 的 instructions 函数、LangGraph 的 node 级 prompt 注入

4

思考链可控化

Claude Extended Thinking、DeepSeek R1 reasoning_content、Gemini thinking mode——思维链从"提示词诱导"变为"API 参数控制"

5

Multi-Agent 即 Service

Agent 框架从实验走向生产，CrewAI Cloud、LangGraph Platform 将多 Agent 协作封装为可部署的服务

6

Safety & Alignment 内建

Guardrails (OpenAI)、Safety Settings (Gemini)、Constitutional AI (Claude)——安全不再是 prompt 里的一句话，而是架构级设计
